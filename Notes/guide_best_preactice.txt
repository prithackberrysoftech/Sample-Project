4. Django ORM & Models (Think ahead)
Rules seniors follow:


Always add related_name


Always think about indexes


Never trust default on_delete


class Order(models.Model):
    user = models.ForeignKey(
        User,
        on_delete=models.PROTECT,
        related_name="orders",
        db_index=True
    )

Why:


Prevents accidental deletes


Faster queries


Clean reverse access
